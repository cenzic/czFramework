Custom form element with hashPath helper - GET method

Using GET method will have all parameter add to url. If url becomes to large then it could cause problem for browser, and it may be distract. However, it could maintain it state and add be able to book-mark.
<form id="customForm" someAtrr="other" method="get" onsubmit="$.jf.formSubmitHandler(event);" action="@@hashPath('simpleModel')">

Using post method to hide all submit input value. If users refresh right away, the state is able to maintain - simulate the browser behavior; however, when come back later, it's no longer hold true.
